利用多target 以及 fastlane 编写iOS打包脚本

发表于 2016-07-10   |  

当你在专心撸代码的时候，测试妹子突然找你说，需要什么什么环境的包的时候，简直想打死她的心都有。然后只能停下手上的活，切分支，改环境，打包，有时候还要上传到蒲公英或者 fir。为了能够用笑脸来迎接测试妹子不时的问候，于是决定写一个 iOS 项目的自动打包脚本。

首先，让我们先理一下需求。

  1. 更改项目配置（例如 host、名字、bundle Id、team、版本号等）
  2. 打包生成 .ipa 文件
  3. 根据需要，将ipa上传到指定的分发平台（蒲公英、fir 等）

接下来我们就来一步一步实现这些需求，完整代码会在文后贴出来。

更改项目配置

对于这个需求，我想到了两个解决方案。

  1. 利用 shell 脚本，直接修改文件
  2. 创建额外的 target，在 target 中配置环境，切换 target 以更改项目配置

权衡之下，我最后选择了方案2。
主要有三个理由。

  1. 如果使用 shell 命令修改文件，会导致脚本与代码耦合非常严重。开发过程中的更改，很有可能导致打包脚本失效
  2. 开发过程中，往往也会需要切换环境，选择 target 来切换环境，会运行脚本更加方便
  3. fastlane 脚本可以非常方便的指定 target 进行打包
  4. 我好像不怎么会 shell …

当然这个方案也有缺点，比如要在代码里根据 target 判断逻辑，或者要防止自己眼瞎，没有正确选择当前 target 之类的。读者也完全可以采用第一种方案，只是不在本文中展开了。

创建 target

创建 target 的方式主要有两种，一种是建立全新的target，另外一种就是复制目前的target，并修改。
推荐大家采用第二种方式，操作方法如下图所示。

在弹出框中，选择Duplicate Only，然后会生成名字为 yourProjectName copy 的 target，以及 yourProjectName copy-Info.plist两个文件。你可以根据需要，对这两个文件重新命名。当你修改 target 的名字时，请记得通过 scheme manager，修改下 scheme 的名字，保持统一。修改 yourProjectName copy-Info.plist 的名字时，会导致新创建的 target 找不到 info.plist，所以请记得重新在 target 中添加回来。然后根据需要，编辑新创建的 target 以及 info.plist，比如修改 bundle Id、name、icon 等。

在代码中判断当前 target

区分的方法有很多。

  1. 根据项目的 bundle Id 来区分，
  2. 根据预定义编译器宏
  3. 通过往各个 target 的 info.plist 里面插入特定的值等等

需要提醒大家的是，如果以后需要修改 target 或者 info.plist，请务必记得把所有的 target 都修改掉。
好的，这样我们就实现了第一个需求。

编写 fastlane 脚本自动打包

fastlane 实际上可以做非常非常多的事情，打包项目只是它其中一个很小的功能而已，这里有列出了所有的 action，像自动生成截图、上传 appstore 等等，大家可以根据需要自行查阅。

安装 fastlane

安装前请确保你已经安装了最新的 Xcode command line tools，通过在终端中执行下面的命令你可以进行检查。

xcode-select --install

然后通过下面的命令，安装 fastlane 以及 gym。由于众所周知的网络原因，最好将 gem 源替换为淘宝源。

sudo gem install fastlane --verbose
sudo gem install gym --verbose

安装完成后，在终端中，进入到iOS项目的根目录，使用 fastlane init 进行初始化。完成之后，项目中，会自动生成 fastlane 文件夹，里面有两个文件，Appfile 和 Fastfile。因为我不需要使用与 appstore 相关的功能，所以不会涉及到 Appfile。

编写脚本

打开 Fastfile，可以看到预定义了很多的 lane，采用的是 ruby 语法。
gym 是一个封装好的打包方法，通过编写 archive方法对它进行再一次封装，方便后续的调用。

def archive(path, name, scheme, method)
    gym(
        export_method: method,
        scheme: scheme,
        clean: true,
        output_directory: path,
        output_name: name,
        configuration: 'Release',
        include_symbols: false,
        include_bitcode: true,
        # use_legacy_build_api: true # 新版本XCode api不稳定
    )
end

path：导出包的路径
name：导出 ipa 的名字，这里有个坑。就是不需要给名字加上 .ipa 后缀，而且就算你加上去了，gym 也会修改这个变量，把 .ipa 去掉
scheme：指定打包的 scheme，用来指定在上文创建的 target
method：指定的打包方法，包括 “ad-hoc” 、”enterprise”、”appstore” 等

include_symbols 以及 include_bitcode 分别对应于下图的两个选项，请根据需要自行选择。

use_legacy_build_api 参数，默认为 false，如果设置为 true，就会使用旧版的打包 api。如果当你运行脚本，打印出某种错误的话，可能就是这个的原因，可以尝试把注释打开，选择旧版 api 再试一次。之前我是必须要设置未 true 的，否则会报错，现在莫名其妙又可以用了，所以没有办法截图了。

好的，编写完方法以后，就可以开始编写我们的打包 lane 了。

desc "打包 type 1: 正式, 0: 测试"
lane :archive do |options|

      time = Time.new.strftime("%Y-%m-%d-%H:%M:%S")
      path = "./fastlane/" + time

    type = options[:type]
    if type == '1'
        name = ‘正式'
        archive(path, name, ‘yourTargetName', 'appstore')
    elsif type == '0'
        name = '测试’
        archive(path, name, 'yourTargetName', 'ad-hoc')
    else 
        puts "type只能是 '0'或者'1'"
    end    

end

这样就完成了打包脚本编写，在终端中进入到项目根目录，就可以通过 fastlane archive type:1 来调用打包脚本了。

上传 ipa 到蒲公英

利用 shenzhen 将打好的包，通过命令，将上传到蒲公英
首先用 gem 安装好 shenzhen

gem install shenzhen —verbose

上传 ipa 的命令如下:
注意：必须先在蒲公英上创建相同 bundle Id 的 app 之后，才能通过命令上传。

ipa distribute:pgyer -f path/to/ipa -u USER_KEY -a APP_KEY

USER_KEY 和 APP_KEY 可以在蒲公英上查到。

上传 ipa 到 fir

利用 fir-cli 将打好的包，通过命令，将上传到蒲公英
首先用 gem 安装好 shenzhen

gem install fir-cli —verbose

上传 ipa 的命令如下:

fir publish path/to/ipa -T YOUR_FIR_TOKEN

YOUR_FIR_TOKEN 可以在 fir 上查到。

在 Fastfile 中调用 shell 命令

通过 sh command 的形式，可以调用 shell 命令。不过 sh 命令有一个坑，使用 scp 命令来上传文件时，会调用失败，所以建议大家使用 ruby 自带的 system 命令来调用 shell 脚本。 

system "echo HelloWorld"

完整 Fastfile 代码

fastlane_version "1.97.2"

default_platform :ios

def archive(path, name, scheme, method)
    gym(
        export_method: method,
        scheme: scheme,
        clean: true,
        output_directory: path,
        output_name: name,
        configuration: 'Release',
        include_symbols: false,
        include_bitcode: true,
        # use_legacy_build_api: true # 新版本XCode api不稳定
    )
end

platform :ios do

    desc "打包 type 1: 正式, 0: 测试"
    lane :archive do |options|

          time = Time.new.strftime("%Y-%m-%d-%H:%M:%S")
          path = "./fastlane/" + time

        type = options[:type]
        if type == '1'
            name = '正式'
            archive(path, name, 'yourTargetName', 'appstore')
        elsif type == '0'
            name = '测试'
            archive(path, name, 'yourTargetName', 'ad-hoc')
            # 上传至蒲公英
            system 'ipa distribute:pgyer -f ./' + time + '/' + name + '.ipa -u your_user_key -a your_app_key'
            # 上传至fir
            system 'fir publish ./' + time + '/' + name + '.ipa -T your_token'
        else 
            puts "type错误, type == '0' : 测试, type== '1' : 正式"
        end    

    end

end
