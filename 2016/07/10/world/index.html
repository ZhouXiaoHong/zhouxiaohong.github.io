fastlane持续集成脚本编写

每次当你在专心撸代码的时候，测试姐姐突然找你说，线上出了个问题，打个测试包或者预发包的时候，简直想打死她的心都有。然后只能停下手上的活，切分支，改环境，打包，有时候还要上传到蒲公英啊或者fir啊什么的。
为了能够用笑脸来迎接测试姐姐不时的问候，于是下定决心研究了下iOS项目的自动打包脚本。
首先，让我们先理一下现在的需求。

  1. 方便的更改项目配置（IP地址、名字、bundle Id、team、版本号等）
  2. 方便的打包，并生成ipa
  3. 根据需要，将ipa上传到指定的分发平台（蒲公英、fir等）
  4. 由于公司目前采用React-Native框架，因此打包完成之后，还需要上传main.jsbundle到服务器
    其实仔细一看，好像并不是很麻烦嘛。接下来我们就来一个一个实现这些需求。

更改项目配置

这个问题我想到了两个解决方案。

  1. 利用shell脚本的sed命令，来修改文件。
  2. 创建额外的target，在target中配置环境。

权衡之下，我最后选择了方案2。
主要有三个理由。

  1. 如果使用sed命令修改文件，会导致脚本与代码耦合非常严重。开发中如果更改了项目的某些配置文件，比如引入了新的环境变量，就有可能导致打包脚本失效。（我才不会告诉你，是因为我不太懂shell命令呢。。。）
  2. 开发过程中，往往也会需要切换环境，利用切换target来切换环境，会比跑脚本快很多。
  3. fastlane脚本可以非常方便的指定打包的target进行打包

当然创建额外的target也会缺点，比如为了达到自动切换配置的目的，要在代码里多写几个if-else啊，或者要防止自己眼瞎，没看仔细当前target就运行啊之类的。当然你也完全可以采用第一种方案，只是不在本文中展开了。不过需要注意一下，在Mac上（iOS开发应该用都用的mac吧？）使用sed命令和linux上会有一定区别。

sed -i  ’s/hello/world/g’ hello.text

上面这行代码，可以在linux上运行，将找到的hello替换为world。但是如果在Mac上，你会发现这行代码会报错。原因在Mac上，sed命令直接操作文件的时候，必须指定备份的格式。

sed -i '' ’s/hello/world/g’ hello.php

即使你不需要备份，你也要向上面的代码一样，带上一对引号。

还有下面插入文本的命令

sed -i '' ' 1i\ hello ' hello.php

比如上面这行代码，往文件中插入hello，在linux下可以运行，但是在Mac上，同样也会报错。如果使用sed命令往文件中插入文本时，必须将要插入的文本独立为一行。
关于使用如何使用shell脚本更改项目配置，不展开了，剩下的请自行研究。

让我们回归正题。那么既然选择使用多个target来管理环境，那么第一步就是要创建新的target。创建的方式主要有两种，一种是建立全新的target，另外一种就是复制目前的target，并修改。
推荐大家采用第二种方式，操作方法如下图所示。

！！！！！！

在弹出框中，选择Duplicate Only。这样你就可以获得XXXX copy的target了，以及一个XXXX copy-Info.plist。你可以根据需要，对这两个文件重新命名。当你修改target的名字，请记得通过scheme manager，修改下scheme的名字，保持统一。
修改XXXX copy-Info.plist的名字时，会导致新创建的target找不到info.plist，所以请记得重新在target中添加回来。
然后就请根据你的需要，编辑新创建的target，比如修改bundle Id、team、name之类的。推荐大家顺便改一下应用图标，可以让你自己或者测试妹子方便的区分应用的环境。
接下来创建好了新的target之后，还有意见非常重要的事情要做。就是如何在代码里根据target的不同，来配置环境呢。方法有很多，比如，根据项目的bundle Id来区分，或者预定义编译器宏，或者通过往各个target的info.plist里面插入特定的值等等，请自行选择，就不展开讲了。
好的，这样我们就实现了第一个需求。

利用fastlane脚本自动打包

fastlane实际上算是一系列脚本的集合，打包只是它其中一个很小的功能而已，这里有它目前集成的所有的action，像自动生成截图、上传appstore等等，大家可以自行查阅。
安装前请确保你已经安装了最新的Xcode command line tools，通过下面的命令你可以检查一下。

xcode-select --install

然后通过下面的命令，安装fastlane以及gym。当然由于众所周知的网络原因，最好将gem源替换为淘宝源。

sudo gem install fastlane --verbose
sudo gem install gym --verbose

安装完成后，在终端中，进入到iOS项目的根目录，使用fastlane init进行初始化。完成之后，项目中，会自动生成fastlane文件夹，里面有两个文件，Appfile和Fastfile。因为我不需要上传appstore的功能，所以不会涉及到Appfile。打开Fastfile，可以看到预定义了很多的lane，然而我都不需要。利用gym，我们并不需要编写太多的代码，首先定义一个打包方法，这个脚本采用的是ruby语法。

def archive(path, name, scheme)
    gym(
        export_method: "ad-hoc",
        scheme: scheme,
        clean: true,
        output_directory: path,
        output_name: name,
        configuration: 'Release',
        include_symbols: false,
        include_bitcode: true,
        # use_legacy_build_api: true # 新版本XCode api不稳定
    )
end

path：导出包的路径
name：导出ipa的名字，这里有个坑。就是不需要给名字加上.ipa后缀，而且就算你加上去了，gym也会帮你把它去掉。
scheme：指定打包的scheme，用来选择在上文配置的target
export_method：指定的打包方法，包括”ad-hoc” 、”enterprise”、”appstore”等

然后要分别解释下gym中的三个配置。
include_symbols以及include_bitcode分别对应于下图的两个选项，请根据需要自行选择。
use_legacy_build_api参数，默认为false，如果设置为true，就会使用旧版的打包api。如果当你运行脚本，打印出如下的错误的话，就请把注释打开。

好的，编写完方法以后，就可以开始编写我们的打包lane了。

desc "打包 type 1: 正式, 0: 测试, -1: 预发"
lane :archive do |options|

      app_name = nil
      time = Time.new.strftime("%Y-%m-%d-%H:%M:%S")
      path = "./fastlane/" + time

    type = options[:type]
    if type == '1'
        name = ‘正式'
            archive(path, name, ‘yourTargetName')
    elsif type == '0'
        name = '线下’
            archive(path, name, 'yourTargetName')
    else 
        name = '预发'
            archive(path, name, 'yourTargetName')
    end    

end

这样就完成了打包脚本，当你进入到项目根目录的时候，就可以通过fastlane archive type:1来调用打包脚本了。

上传ipa到蒲公英

利用shenzhen将打好的包，通过命令，将上传到蒲公英
首先用gem安装好shenzhen

gem install shenzhen —verbose

上传ipa的命令如下:
注意：必须先在蒲公英上创建相同bundle Id的app之后，才能通过命令上传。

ipa distribute:pgyer -u USER_KEY -a APP_KEY

USER_KEY和APP_KEY可以在蒲公英上查到。

可以在上文写的lane中，通过 sh command的形式，调用shell脚本。不过sh命令也有一个坑，就是如果使用scp命令来上传文件的话，会调用失败，所以建议大家使用ruby自带的system命令来调用shell脚本。 

system command

上传ipa到fir

利用fir-cli将打好的包，通过命令，将上传到蒲公英
首先用gem安装好shenzhen

gem install fir-cli —verbose

上传ipa的命令如下:

fir publish path/to/application -T YOUR_FIR_TOKEN

YOUR_FIR_TOKEN可以在fir上查到。
